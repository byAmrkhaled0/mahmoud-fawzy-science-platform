# تقرير تطوير منصة العلوم — الإصدار 53

## نطاق العمل

تم تطوير الإصدار 52 إلى الإصدار 53 مع الحفاظ على نظام تحصيل الاشتراك داخل السنتر فقط. لم تتم إضافة بوابة دفع إلكتروني أو تحويلات مالية داخل الموقع.

## التحسينات المنفذة

### 1. توافق كامل مع الموبايل

- إصلاح تعارض CSS كان يجعل القائمة الجانبية في لوحة الإدارة تشغل ارتفاع شاشة كاملة قبل المحتوى.
- تحويل القائمة الجانبية على الشاشات الصغيرة إلى Drawer ثابت لا يدفع الصفحة لأسفل.
- إضافة شريط تنقل سفلي مناسب للمس داخل لوحة الإدارة.
- تحسين النماذج والبطاقات والجداول والأزرار ومساحات اللمس على الشاشات الصغيرة.
- تنظيم إجراءات الطالب الثانوية داخل قائمة «المزيد من الإجراءات» لتقليل الزحام.
- منع التمرير الأفقي على عرض 390 بكسل في الاختبار النهائي.

### 2. تثبيت المنصة على الهاتف PWA

- تحديث `site.webmanifest` مع `id` و`scope` و`start_url` ووضع `standalone`.
- إضافة أيقونات 192 و512 بكسل وأيقونة Maskable.
- تحديث Service Worker إلى Cache الإصدار 53.
- على Android وChrome أو Edge: زر «تثبيت المنصة» يستخدم نافذة التثبيت الأصلية عند توافرها.
- على iPhone وiPad: يعرض الزر خطوات Safari الصحيحة: مشاركة ← إضافة إلى الشاشة الرئيسية ← إضافة.
- إخفاء زر التثبيت تلقائيًا بعد تشغيل المنصة في وضع التطبيق.

> التثبيت الفعلي يحتاج نشر الموقع عبر HTTPS. متصفحات iPhone لا توفر نفس نافذة `beforeinstallprompt` الموجودة في Android، لذلك يتم عرض تعليمات الإضافة إلى الشاشة الرئيسية.

### 3. إعادة تنظيم بيانات Firebase

- إيقاف الاعتماد على مستند واحد كبير باسم `settings/siteData` للحفظ المستمر.
- حفظ الطلاب والحضور والدرجات والتسميع والامتحانات في Collections منفصلة.
- ترحيل البيانات القديمة تلقائيًا عند أول دخول إداري بعد نشر الإصدار الجديد.
- تقليل الكتابات غير الضرورية عبر مقارنة بصمة البيانات قبل المزامنة.

### 4. حماية النماذج العامة

- إلغاء الكتابة المباشرة العامة للحجوزات والتقييمات في Firestore.
- تمرير الحجز والتقييم من خلال Cloud Functions فقط.
- الإبقاء على Rate Limiting والتحقق من المدخلات على الخادم.
- تحديث Firestore Rules لمنع إنشاء الحجوزات والتقييمات مباشرة من المتصفح.

### 5. العام الدراسي والترم

- إضافة العام الدراسي والترم إلى بيانات الطالب والحضور والامتحان.
- إضافة فلاتر العام الدراسي والترم داخل لوحة الطلاب.
- إضافة «تانية ثانوي» إلى الصفوف المتاحة.
- منع اختلاط سجلات السنوات الدراسية المختلفة عند استخدام البيانات الجديدة.

### 6. اشتراك السنتر

- لا يوجد دفع إلكتروني داخل المنصة.
- القسم مخصص فقط لتسجيل: «تم الدفع في السنتر» أو «لم يدفع».
- إضافة تصدير CSV لحالات اشتراك السنتر مع الشهر والعام الدراسي والترم.

### 7. الاستيراد والتصدير

- استيراد الطلاب من CSV أو Excel.
- دعم أعمدة الاسم والهاتف وهاتف ولي الأمر والصف والمجموعة والشهر والعام الدراسي والترم والملاحظات.
- تصدير الطلاب والحضور ودرجات الامتحانات واشتراكات السنتر بصيغة CSV.

### 8. النسخ الاحتياطي والاستعادة

- نسخة احتياطية سحابية مجدولة يوميًا الساعة 02:30 بتوقيت القاهرة.
- إنشاء نسخة يدوية وعرض النسخ وتنزيلها من لوحة الإدارة.
- إنشاء نسخة أمان إضافية قبل تنفيذ الاستعادة.
- حماية الاستعادة بكلمة تأكيد `RESTORE-V53`.
- حذف الطالب عبر Cloud Function مع أرشفة بياناته قبل حذف السجلات المرتبطة.

> النسخ المجدول يستخدم Cloud Scheduler، وقد يحتاج إلى خطة Firebase تدعم الدوال المجدولة والفوترة المرتبطة بها.

### 9. الصلاحيات والحسابات

- تقييد حساب المساعد للأقسام التشغيلية المحددة.
- منع المساعد من حذف الطلاب أو تغيير أكواد الطالب وولي الأمر.
- إضافة زر إعادة تعيين كلمة مرور حساب فريق العمل.
- تفعيل MFA نفسه يحتاج تشغيل Identity Platform وإعداده من Firebase Console.

### 10. الامتحانات والمراقبة

- تحديد الصف والمجموعة والعام الدراسي والترم لكل امتحان.
- تحديد تاريخ ووقت الفتح والإغلاق.
- التحقق من أهلية الطالب ووقت الإتاحة داخل Cloud Function.
- إضافة واجهة لسجل أخطاء المتصفح `client_errors` لتسهيل اكتشاف الأعطال.

- فحص JavaScript وJSON وروابط ملفات HTML: ناجح.
- فحص قواعد الأمان ومسارات النماذج العامة: ناجح.
- فحص Manifest وService Worker وأيقونات التثبيت: ناجح.
- اختبار محاكاة لوحة الإدارة على عرض 390×844 بكسل: لا يوجد تمرير أفقي، والمحتوى يبدأ من أعلى الشاشة، والقائمة تعمل كـDrawer.
- اختبار الصفحة الرئيسية على عرض موبايل: لا يوجد تمرير أفقي، وشريط الموبايل وزر التثبيت موجودان.

## ترتيب النشر الصحيح

من داخل مجلد المشروع وبعد تسجيل الدخول إلى Firebase CLI:

```powershell
npm install
npm run firebase:deploy:backend
npm run firebase:hosting
```

أو استخدم الأمر الموحد:

```powershell
npm run firebase:deploy:all
```

ترتيب نشر الـBackend أولًا مهم لأن النسخة الجديدة تمنع الكتابة المباشرة للحجز والتقييم، وتعتمد على Cloud Functions الجديدة.

## فحص بعد النشر

1. افتح الموقع من رابط HTTPS عادي، وليس من ملف محلي.
2. امسح Cache الموقع القديم أو نفذ تحديثًا إجباريًا مرة واحدة عند الانتقال من الإصدار 52.
3. سجل دخول المدرس وتأكد من ظهور الطلاب الحاليين بعد الترحيل.
4. أنشئ حجزًا تجريبيًا وتقييمًا تجريبيًا.
5. جرّب إضافة طالب وحضور ودرجة ثم أعد تحميل الصفحة.
6. أنشئ نسخة احتياطية يدوية وتأكد من ظهورها في قائمة النسخ.
7. على Android اضغط «تثبيت المنصة» من Chrome.
8. على iPhone افتح Safari واضغط زر التثبيت ثم اتبع خطوات «إضافة إلى الشاشة الرئيسية».

## حدود الاختبار

تم اختبار البناء والكود والواجهات بمحاكاة موبايل محلية. الاختبار النهائي المتصل بخدمات Firebase الفعلية، بما يشمل Authentication وCloud Functions وFirestore وCloud Scheduler، يحتاج نشر المشروع واستخدام بيانات اعتماد Firebase الخاصة بالمنصة.
